# Docs Overview

Use these files as the canonical documentation structure for a new project.

- `context.md` — fastest recovery entrypoint
- `current-state.md` — current execution state and queue
- `current-system.md` — implemented system behavior only
- `current-product.md` — implemented product behavior only
- `future-roadmap.md` — not-yet-implemented direction
- `glossary.md` — canonical terms
- `history/changelog.md` — completed outcomes only

Process docs live in `docs/process/`.
Execution plans live in `docs/plans/`.
